## 2. Architecture Deep Dive

### 2.1 Three-Layer Design

```
┌─────────────────────────────────────────────────┐
│                  CLI (cli.py)                    │
│         Unified entrypoint + --domain dispatch   │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│              Core Engine (core/)                 │
│  interfaces.py │ domain_registry.py │ lifecycle  │
│  tracker.py    │ utils.py                        │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│           Domains (domains/*)                    │
│  hndsr_vr/ (CV)        │ nlp_lm/ (NLP)          │
│  + Backwards shims (hndsr_vr/)                   │
└─────────────────────────────────────────────────┘
```

**Grade: ✅ A-**

The separation is clean and real. Core never imports domain code. Domains only import from core. The shim layer preserves backwards compatibility. The auto-discovery via `domain.yaml` manifests is elegant.

**One nitpick:** `REPO_ROOT = Path(__file__).resolve().parents[3]` in `core/utils.py` is fragile — it assumes a fixed nesting depth. If the package is installed via pip into a site-packages directory, this breaks.
